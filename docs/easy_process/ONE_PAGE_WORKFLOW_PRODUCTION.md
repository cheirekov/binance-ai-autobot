## The first 5 questions at batch start
1. Which production capability lane matters most now?
2. Does the active ticket move that capability forward?
3. Is the latest evidence fresh enough for code?
4. Is the next move runtime patch, validation-only, or promotion work?
5. What does this batch do to move the bot toward bounded production readiness?

## Required decision modes
- `NO_CODE`
- `VALIDATION_ONLY`
- `PATCH_ALLOWED`
- `PROMOTION_PACKET_ONLY`

## Hard program rules
- No stale-evidence runtime patch.
- No “self-learning” live mutation.
- No news/headline prose directly drives orders.
- No ticket work that ignores the capability roadmap.
- No claim of adaptation without multi-window evidence.
